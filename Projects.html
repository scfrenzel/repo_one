Web Applications

IT & IS Project Dashboard

I built this application as a way to manage all projects related to the IT and IS departments. This application has all the basic requirements you'd expect from management software with a few extra features which include an Overview, Task assignment section and Configurations section.

The overiew allows a user to quickly view how many projects are assigned to them, how many projects are behind schedule, and total projects by department. Each graph is responsive, and hovering over them provides additional detail.

Here you can see the many points of interest regarding tracking the progress of a project.

Here's the template for creating a new project.

In-House IT HelpDesk

This project was a two-part venture, the first being the interface for the end users, the second was the ticket management portion. This new system uses Active Directory to leverage user-to-ticket mapping. This aids in locating users, emailing upates, providing up-to-date contact information, and identifying problematic PCs. The application used to submit a ticket was desinged with simplicity in mind (typically requiring only 5 clicks to submit a ticket). The management section is used to work on and complete tickets.

If the user is not logged into their PC, or someone is submitting a ticket on their behalf, they can enter contact information into this modal and the ticket will then be tied to that information instead of the current AD / Windows user.

This is the summary page just before a ticket is submitted. From here the user can verify the correct information and submit a new ticket.

Here's a view of the management side. A technician can enter pertinent details, provide resolution notes, update the time spent, and further categorize the ticket.

Automated Order Creator

This application was built to automate a frequent and time consuming process. When a user receives an email about creating new orders, she would upload the Excel sheet containing orders, the application would parse then display the orders with additional details pulled from our ERP system. Once the orders have been approved, she clicks the "Create" button and the application creates the orders using the API provided by our ERP system.

.NET Applications

EDI Automation

This application handles EDI communication from one of our customers. As a daily task, this console application builds EDI files using invoice data from our ERP system, and submits them via FTP. The counterpart to this application is another console application what waits for acknowledgements, downloads them when they are available, parses the file and saves the data to a database. These two independent automated processes remove the need human interaction for processing payments with EDI communication.

Disk Space Monitor

This application runs as a monitoring service which periodically checks the available disk space of several mission critical servers. Should one of the servers be low on that resource, an email is sent to the IT team informing them of the issue at hand. This was built as a temporary solution to prevent server functionality failure on a company wide scale.
